[프로젝트] LEAFYER : 식물 커뮤니티 (진행)

[개요]

  • 식집사들을 위한 커뮤니티로 게시글 작성, 식물 거래 정보 조회, 1대1 채팅 기능을 제공
  • 팀 구성: FE 3명 / BE 4명
  • 담당 역할: 디자인 및 FE 개발자
  • 기술: react, typescript, emotion, mobx, github / figma /

[목표]

  • 기획 및 디자인부터 시작하여 개발하는 프로젝트 경험
  • 개인의 성장보다는 프로젝트를 우선하는 마인드로 참여

[경험]

  • 1. 디자이너의 입장에서 개발자와 협업 / 피그마를 활용하여 디자인 시스템

    도입 어도비 XD로 디자인을 하며 한 달 정도 기획과 디자인을 하였지만, 협업의 어려움으로 인해 과감히 피그마로 다시 시작하였습니다. 팀원의 피드백을 받고 전체적으로 색감을 교체, 페이지의 기능도 전체적으로 수정하면서 디자이너의 역할을 수행하며 개발자와의 협업을 경험했습니다.

  • 2. 개발자의 입장에서 디자인 적용 / 피그마와 깃허브를 연동하여 디자인

    토큰 적용 피그마의 플러그인을 활용하여 디자인 시스템을 json 파일로 변환하여 깃허브에 커밋하는 작업을 진행했습니다. 이 json 파일을 이용하여 버튼 컴포넌트에 직접 적용을 하였습니다. 디자인 시스템을 그대로 적용하려니 하나의 컴포넌트에 여러 개의 파일이 필요하고, 버튼의 사이즈 등을 강제하는 등 활용성이 떨어지는 여러 문제점들이 생겼습니다. 기본적인 색상값, 폰트, 공통 스타일에 대해서만 피그마와 연동하여 의존성을 줄이고, 이벤트보다는 emotion을 이용하여 코드를 작성하였습니다. 아토믹 디자인 시스템을 도입하여 컴포넌트 내에 아톰 컴포넌트들로 구성이 되어 있는 형태여서 오히려 피그마와의 연동이 문제로 작용하였던 것입니다. 이를 동료와의 커뮤니케이션으로 개선하는 과정에서 가독성이 좋은 코드, 리액트의 컴포넌트에 대해서도 학습할 수 있었습니다.
